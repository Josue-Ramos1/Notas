# 🐧 Todo sobre Linux

Linux es el sistema operativo más usado en el área de seguridad actualmente. Es un sistema de **código abierto**, lo que significa que cualquiera puede usar, compartir y modificar su código.

Su historia comienza en los 90, cuando **Linus Torvalds** creó el **kernel de Linux** como mejora a UNIX, mientras **Richard Stallman** desarrollaba el proyecto **GNU** con la misma filosofía. Al combinar el kernel de Torvalds y el sistema GNU, nació el Linux que conocemos hoy.

<Aside type="tip">
Gracias a la licencia pública GNU, existe una gran comunidad que mejora continuamente Linux. Actualmente hay más de **600 distribuciones**, cada una adaptada a necesidades específicas.
</Aside>

En seguridad básica, Linux se usa para revisar logs, verificar accesos, analizar incidentes y ejecutar herramientas forenses y de pruebas de penetración. Su flexibilidad lo hace indispensable para cualquier analista.

---

# 🏗️ Arquitectura de Linux: visión general

Al igual que un edificio tiene estructuras bien definidas, Linux está compuesto por varios elementos clave:

<Steps>
1. <strong>Usuario</strong>: Interactúa y ejecuta comandos. Linux permite múltiples usuarios simultáneos (multiusuario).
2. <strong>Aplicaciones</strong>: Programas específicos (navegadores, editores). Se instalan mediante gestores de paquetes. Ejemplo: <code>nano</code>.
3. <strong>Shell</strong>: Interfaz CLI que interpreta los comandos y devuelve resultados.
4. <strong>FHS (File Hierarchy Standard)</strong>: Estructura que organiza los archivos, facilitando búsquedas y accesos.
5. <strong>Kernel</strong>: Núcleo que administra memoria, procesos y se comunica con el hardware.
6. <strong>Hardware</strong>: Componentes físicos como CPU, teclado, disco. El kernel interactúa con ellos para ejecutar instrucciones.
</Steps>

<Aside type="note">
El kernel es el "motor" del sistema y el elemento más crítico para el rendimiento y la seguridad.
</Aside>

# 💽 Distribuciones de Linux

## 🚀 Distribuciones y su importancia en ciberseguridad

Linux no tiene una única versión: existen **muchas distribuciones** (o *distros*), cada una con herramientas y configuraciones distintas.

<Aside>
El kernel de Linux es como un motor, y cada distribución es un vehículo construido sobre ese motor. Algunas son deportivas, otras familiares; cada una se adapta a una necesidad.
</Aside>

Distribuciones conocidas y sus orígenes:

* **Ubuntu** y **Kali Linux** derivan de **Debian**.
* **CentOS** viene de **Red Hat**.
* **SUSE** viene de **Slackware**.

Cada distribución puede incluir diferentes entornos gráficos, programas y configuraciones. Entender cuál usas es clave para eficiencia y compatibilidad.

# 📦 Administradores de paquetes

Un **paquete** es una unidad de software que puede incluir aplicaciones o partes de ellas, junto con sus dependencias. Los **administradores de paquetes** permiten instalar, actualizar y gestionar software fácilmente, resolviendo dependencias automáticamente.

## 🌳 Distribuciones y tipos

| Distribución base | Administrador de paquetes | Extensión de archivo |
| ----------------- | ------------------------- | -------------------- |
| Debian            | `dpkg`                    | `.deb`               |
| Red Hat           | `rpm`                     | `.rpm`               |

Ejemplos de nombres de archivo:

* Debian: `nombre_versión-arquitectura.deb`
* Red Hat: `nombre-versión-release_arquitectura.rpm`

---

# 🐚 Shell en Linux

El **shell** es la interfaz que permite comunicarse con el sistema a través de texto. Es una herramienta clave para administradores y analistas de seguridad.

## 🔎 Funciones principales

Permite:

- Ejecutar comandos para interactuar con el sistema.
- Automatizar tareas y conectar aplicaciones.
- Gestionar archivos y procesos.
- Comunicarse directamente con el kernel.

---

## 💬 Importancia en seguridad

El shell facilita operaciones avanzadas y auditorías. Incluso si el sistema puede operar sin shell, es esencial para interacción humana directa.

---

## 🌀 Tipos de shell

El más común en ciberseguridad es **Bash (Bourne Again Shell)**. Otros son `sh`, `zsh` o `fish`. Bash se destaca por ser el más soportado y flexible.

---

# 🔧 Comunicación con el shell

Interactuar con el shell es como conversar: das una orden y recibes respuesta.

## 📥 Entrada estándar (stdin)

La información que introduces desde el teclado. Ejemplo:

```bash
echo hola
```

## 📤 Salida estándar (stdout)

La respuesta del sistema. Ejemplo: tras `echo hola`, el sistema imprime `hola`.

## ⚠️ Error estándar (stderr)

Mensajes cuando ocurre un error. Ejemplo: escribir `eco hola` genera un error porque el comando está mal escrito.

<Aside type="tip">
Entender stdin, stdout y stderr es vital para crear scripts y depurar comandos en seguridad.
</Aside>
