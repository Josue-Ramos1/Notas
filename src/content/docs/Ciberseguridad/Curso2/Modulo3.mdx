# Herramientas SIEM

Las **herramientas SIEM** (Security Information and Event Management) recopilan y analizan datos de registro en tiempo real para ayudar a los analistas de seguridad a identificar amenazas, riesgos y vulnerabilidades.

<CardGrid>
  <Card title="Registros de firewall" icon="shield">
    Registran conexiones entrantes y solicitudes salientes.
  </Card>
  <Card title="Registros de red" icon="globe">
    Muestran conexiones entre dispositivos y servicios.
  </Card>
  <Card title="Registros de servidor" icon="server">
    Registran eventos como inicios de sesión y cambios de credenciales.
  </Card>
</CardGrid>

<Aside>
El monitoreo constante de estos registros permite detectar vulnerabilidades y posibles violaciones de seguridad.
</Aside>

## ¿Qué hacen las herramientas SIEM?

- Recopilan registros de múltiples fuentes.
- Analizan y correlacionan datos en tiempo real.
- Generan alertas automáticas.
- Centralizan el almacenamiento para auditorías.

<Aside type="tip">
Requieren configuración y ajuste constante para adaptarse a nuevas amenazas.
</Aside>

## Paneles en herramientas SIEM

<Tabs>
  <TabItem label="¿Qué son?">
    Los **paneles de SIEM** son interfaces gráficas que muestran datos de seguridad mediante gráficos, tablas y métricas clave.
  </TabItem>
  <TabItem label="¿Para qué sirven?">
    Facilitan detectar patrones, tendencias y comportamientos sospechosos rápidamente.
  </TabItem>
  <TabItem label="Ejemplo práctico">
    Si recibes una alerta de inicio de sesión sospechoso, el panel puede mostrar que hubo **500 intentos en 5 minutos** desde ubicaciones inusuales y fuera de horario.
  </TabItem>
</Tabs>

<Aside type="tip">
Además de amenazas, los paneles pueden incluir métricas como disponibilidad, tasa de fallos y volumen de tráfico de red.
</Aside>

## Tipos de SIEM

<CardGrid>
  <Card title="Autoalojado" icon="database">
    Instalado y gestionado por la propia organización. Ofrece mayor control sobre los datos.
  </Card>
  <Card title="En la nube" icon="cloud">
    Gestionado por un proveedor externo. Reduce costos de infraestructura y ofrece escalabilidad.
  </Card>
  <Card title="Híbrido" icon="layers">
    Combina nube y local para equilibrar control y flexibilidad.
  </Card>
</CardGrid>

<Aside type="caution">
Cada opción tiene ventajas y desventajas; depende de la sensibilidad de los datos y recursos disponibles.
</Aside>

# Código abierto vs propietario

<Tabs>
  <TabItem label="Código abierto">
    - Gratuitas, desarrolladas colaborativamente.  
    - Código fuente público permite auditoría y mejoras.  
    - Ejemplos: **Linux** (sistema base ampliamente usado) y **Suricata** (detección de tráfico sospechoso).
  </TabItem>
  <TabItem label="Propietarias">
    - Desarrolladas por empresas, acceso restringido al código.  
    - Requieren licencias.  
    - Ofrecen soporte oficial.  
    - Ejemplos: **Splunk®** y **Chronicle SIEM**.
  </TabItem>
</Tabs>

<Aside>
Las herramientas de código abierto pueden ser tan seguras y robustas como las propietarias, gracias a la revisión continua de la comunidad.
</Aside>

## Paneles de Splunk

- **Postura de seguridad**: Eventos en tiempo real para SOCs.
- **Resumen ejecutivo**: Estadísticas para toma de decisiones.
- **Revisión de incidentes**: Cronología visual de eventos sospechosos.
- **Análisis de riesgos**: Identifica usuarios, equipos o IP con mayores riesgos.

## Paneles de Chronicle

- **Estadísticas empresariales**: Muestra IOC y niveles de gravedad.
- **Ingestión de datos y salud**: Verifica registros procesados sin errores.
- **Coincidencias COI**: Amenazas por dominios/IP sospechosos.
- **Principal**: Resumen de eventos y tendencias.
- **Detecciones de reglas**: Incidentes recurrentes para optimizar respuestas.
- **Inicio de sesión de usuario**: Accesos anómalos.

<Aside type="danger">
Tanto Splunk como Chronicle ayudan a detectar y responder incidentes, fortaleciendo la postura de seguridad.
</Aside>
