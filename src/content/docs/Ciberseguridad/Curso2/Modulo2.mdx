# Marcos y Controles de Seguridad

## Marcos de Seguridad

Los marcos de seguridad proporcionan directrices estructuradas para planificar y mitigar amenazas y riesgos, además de facilitar el cumplimiento de regulaciones.

> 🏥 Ejemplo: El sector salud utiliza marcos como HIPAA para proteger datos de pacientes.

## Controles de Seguridad

Son salvaguardas específicas aplicadas para mitigar riesgos. Por ejemplo, la autenticación multifactor (MFA) en hospitales ayuda a cumplir con HIPAA restringiendo accesos indebidos.

## Marcos y Controles Específicos

### Marco de Amenaza Cibernética (CTF)

Diseñado por el gobierno de EE.UU. para unificar la comunicación sobre amenazas. Mejora la respuesta ante ataques mediante taxonomías compartidas.

### ISO/IEC 27001

Un estándar internacional para gestionar la seguridad de la información. Define un Sistema de Gestión de Seguridad de la Información (SGSI), orientado a proteger datos financieros, de empleados y terceros.

## Tipos de Controles de Seguridad

### Controles Físicos

* Cerraduras, vallas, guardias de seguridad
* Cámaras CCTV y tarjetas de acceso

### Controles Técnicos

* Firewalls
* MFA
* Antivirus

### Controles Administrativos

* Separación de funciones
* Gestión de accesos
* Políticas de clasificación de activos

## Tríada CID: Confidencialidad, Integridad y Disponibilidad

### 1. Confidencialidad

Restringe el acceso solo a usuarios autorizados. Aplicación del **principio de privilegio mínimo**.

### 2. Integridad

Asegura que los datos no han sido alterados. Se implementa con **criptografía y hash**.

### 3. Disponibilidad

Los datos deben estar accesibles para usuarios autorizados cuando los necesiten.

> ✅ Un sistema robusto equilibra estos tres pilares para minimizar riesgos.

---

## Marco de Ciberseguridad del NIST (CSF)

Desarrollado por el NIST, este marco global se compone de cinco funciones clave:

### 1. Identificar

Reconocer activos, sistemas y riesgos. Ejemplo: detectar dispositivos no autorizados.

### 2. Proteger

Desarrollar políticas y controles para proteger los recursos.

### 3. Detectar

Identificar incidentes con rapidez. Uso de SIEM y monitoreo continuo.

### 4. Responder

Contención, análisis e informe del incidente.

### 5. Recuperar

Restauración de sistemas y activos críticos tras una infracción.

> 🔐 El NIST también incluye el **SP 800-53**, usado en entornos gubernamentales para proteger la información cumpliendo la tríada CID.

## Principios de Seguridad según OWASP

1. **Minimizar superficie de ataque**
2. **Privilegio mínimo**
3. **Defensa en profundidad**
4. **Separación de funciones**
5. **Seguridad simple**
6. **Corrección eficaz**
7. **Valores seguros por defecto**
8. **Fallo seguro**
9. **Desconfianza de servicios externos**
10. **Evitar seguridad por oscuridad**

> 📌 Estos principios fortalecen la seguridad y reducen vectores de ataque.

## Auditorías de Seguridad

### Objetivo

Revisar controles internos y externos para detectar deficiencias, asegurar cumplimiento y evitar sanciones.

### Factores que afectan

Industria, tamaño de empresa, ubicación y regulaciones vigentes.

### Marcos utilizados

* **NIST CSF**
* **ISO 27000**

### Pasos clave

1. Definir alcance
2. Evaluar riesgos
3. Ejecutar auditoría
4. Crear plan de mitigación
5. Comunicar resultados

> 📊 Las auditorías permiten fortalecer la postura de seguridad mediante informes claros y planes de acción documentados.
