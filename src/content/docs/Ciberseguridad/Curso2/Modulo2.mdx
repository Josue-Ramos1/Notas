## Tríada CID

<Steps>
1. **Confidencialidad**: Limitar el acceso solo a usuarios autorizados (ej. principio de privilegio mínimo).
2. **Integridad**: Proteger datos contra modificaciones no autorizadas (ej. criptografía).
3. **Disponibilidad**: Garantizar que los datos estén accesibles para quien los necesita (ej. acceso remoto seguro).
</Steps>

<Aside>
Mantener equilibrio entre los tres principios es clave para una seguridad sólida.
</Aside>

## Marco de Ciberseguridad del NIST (CSF)

<Aside>
El **NIST CSF** es uno de los marcos más usados globalmente para gestionar riesgos de ciberseguridad. Contiene cinco funciones básicas.
</Aside>

<CardGrid>
  <Card title="Identificar" icon="search">
    Reconocer activos y riesgos.
  </Card>
  <Card title="Proteger" icon="shield">
    Implementar medidas y políticas.
  </Card>
  <Card title="Detectar" icon="alert-circle">
    Descubrir incidentes rápidamente.
  </Card>
  <Card title="Responder" icon="activity">
    Actuar ante incidentes.
  </Card>
  <Card title="Recuperar" icon="refresh-ccw">
    Restaurar servicios y sistemas.
  </Card>
</CardGrid>

<Aside type="tip">
Ejemplo práctico: ante un dispositivo desconocido, se identifica la amenaza, se protege bloqueando el dispositivo, se detectan otras anomalías, se responde investigando y se recuperan archivos afectados.
</Aside>

<Aside>
El **SP 800-53** del NIST también mantiene la tríada CID para sistemas gubernamentales.
</Aside>

## Auditorías de Seguridad

<Steps>
1. **Definir el alcance**: Determinar qué recursos evaluar.
2. **Evaluación de riesgos**: Relacionar amenazas con presupuesto y regulaciones.
3. **Realizar la auditoría**: Revisar controles técnicos, físicos y administrativos.
4. **Plan de mitigación**: Diseñar acciones para reducir riesgos.
5. **Comunicar resultados**: Elaborar informe con hallazgos y mejoras.
</Steps>

<Aside type="tip">
Marcos como **NIST CSF** e **ISO 27000** ayudan a preparar auditorías internas y externas.
</Aside>

<Aside type="danger">
Las auditorías son clave para evitar sanciones y proteger datos contra amenazas.
</Aside>
