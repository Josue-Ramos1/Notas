# Herramientas importantes de ciberseguridad

El kit de herramientas de un analista de seguridad de nivel básico puede variar según las necesidades de cada organización. Es esencial conocer las herramientas estándar del sector y demostrar la capacidad de aprender nuevas tecnologías en entornos reales de trabajo.

### Registros y monitoreo

Los **registros** documentan eventos relevantes en los sistemas, como inicios de sesión o accesos a servicios web. Son fundamentales para identificar vulnerabilidades e investigar incidentes.

### Herramientas SIEM

Las plataformas **SIEM (Security Information and Event Management)** recopilan y analizan registros de múltiples fuentes para:

- Detectar amenazas en tiempo real.
- Generar alertas automatizadas.
- Presentar datos mediante paneles visuales.

> ☁️ Estas herramientas pueden ser implementadas en la nube o en servidores locales (_on-premise_), dependiendo de la experiencia y recursos del equipo de seguridad.

### Analizadores de protocolos de red

También conocidos como **sniffers** o **rastreadores de paquetes**, estas herramientas capturan y analizan el tráfico de red, facilitando la identificación de actividad sospechosa.

> 🧪 Más adelante en el programa, se practicará con herramientas específicas de este tipo.

### Manuales y estrategias

Los **manuales de estrategias** detallan procedimientos para la respuesta a incidentes y son esenciales en investigaciones forenses. Existen dos tipos clave:

- **Libro de jugadas de la cadena de custodia:**  
  Documenta el manejo, posesión y control de las pruebas digitales.

- **Manual de protección y preservación de evidencias:**  
  Prioriza la conservación de datos frágiles y volátiles según el **orden de volatilidad**.

> ⚠️ Un manejo inadecuado de las pruebas puede invalidarlas legalmente. Para preservar la integridad, siempre se deben trabajar con copias de los datos.

# Conocimientos y habilidades básicas de ciberseguridad

## Herramientas técnicas

En ciberseguridad, las herramientas permiten automatizar tareas, detectar amenazas y proteger los sistemas. A continuación, se describen algunas categorías clave:

### Programación y bases de datos

- **Python**: Se utiliza para automatizar tareas rutinarias y desarrollar scripts que agilizan análisis o respuestas.
- **SQL**: Permite consultar bases de datos para extraer información útil en investigaciones o auditorías.

### Sistemas operativos

- **Linux**, **macOS** y **Windows** son las plataformas principales.
- Linux destaca por su enfoque en línea de comandos y su naturaleza de código abierto, que permite personalización.

### Seguridad de aplicaciones web

Las **vulnerabilidades web** permiten a los atacantes robar información o insertar malware. Para mantenerse actualizado:

> 🔎 Revisa regularmente el **OWASP Top 10**, una lista reconocida de riesgos críticos en aplicaciones web.

### Protección de endpoints

- **Antivirus**: Escanean y eliminan amenazas de malware utilizando firmas digitales o análisis heurístico.
- **IDS (Intrusion Detection Systems)**: Monitorean el tráfico en busca de comportamientos sospechosos, como intentos de acceso no autorizados.

### Encriptación

- Convierte los datos en un formato ilegible sin la clave correcta.
- Protege la **confidencialidad** de la información frente a accesos no autorizados.
- Se diferencia de la codificación, que busca compatibilidad entre sistemas.

### Pruebas de penetración (Pen Testing)

- Simulan ataques reales para detectar vulnerabilidades.
- Ayudan a **evaluar la postura de seguridad** de redes, aplicaciones y sistemas internos.

# Portafolio profesional en ciberseguridad

Un **portfolio** es una herramienta fundamental para mostrar habilidades, formación y experiencia en el área de seguridad informática. Aporta credibilidad técnica y permite destacar frente a posibles empleadores.

## ¿En qué se diferencia del currículum?

- El **currículum** es un resumen breve.
- El **portfolio** presenta proyectos concretos, documentación y logros específicos.

## Opciones para construir tu portfolio

1. **Carpeta local**  
   Organiza subcarpetas por tema: currículum, herramientas, proyectos.

2. **Google Drive / Dropbox**  
   Ideal para compartir documentos con actualizaciones automáticas en la nube.

3. **Google Sites**  
   Permite crear una página web personalizable para mostrar tu trayectoria de forma profesional.

4. **Repositorio Git (GitHub, GitLab, Bitbucket)**  
   Muestra código, scripts y documentación técnica usando Markdown para claridad.

## ¿Qué incluir?

Durante tu formación o experiencia práctica, puedes documentar:

- Auditorías de seguridad
- Análisis de tráfico de red
- Consultas SQL aplicadas
- Gestión de permisos en Linux
- Documentación de incidentes
- Proyectos con herramientas SIEM o IDS
- Revisión y mejoras de currículum

> 🚫 Recuerda: **no compartas información sensible** ni propiedad de otras organizaciones. Mantén tu portfolio privado hasta que esté listo para ser evaluado.
