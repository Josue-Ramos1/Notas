# Herramientas importantes de ciberseguridad

El kit de herramientas de un analista varía según la organización, pero es esencial conocer las herramientas estándar y demostrar capacidad de aprendizaje continuo.

## Registros y SIEM

Los **registros** documentan eventos en los sistemas (como inicios de sesión y accesos), y son clave para identificar vulnerabilidades y posibles infracciones.

Las herramientas **SIEM** recopilan y analizan datos de registros para monitorear actividades críticas y detectar amenazas. Reducen la carga de trabajo al generar alertas específicas y ofrecen paneles visuales. Pueden alojarse en la nube o on-premise.

## Analizadores de protocolos y manuales de estrategia

Los **analizadores de protocolos de red** (sniffers) registran y analizan el tráfico dentro de la red. Más adelante en el curso se practicarán herramientas comunes.

Los **manuales de estrategias** detallan los procedimientos para responder a incidentes. Incluyen:

<CardGrid>
  <Card title="Libro de jugadas de cadena de custodia" icon="clipboard-list">
    Documenta la posesión y control de pruebas digitales durante una investigación.
  </Card>
  <Card title="Manual de preservación de evidencias" icon="shield-check">
    Prioriza conservar datos digitales frágiles usando el orden de volatilidad.
  </Card>
</CardGrid>

<Aside type="caution">
Un manejo incorrecto de pruebas digitales puede comprometer su validez. Siempre se deben hacer copias y trabajar sobre ellas para preservar la integridad.
</Aside>

# Conocimientos y habilidades básicas de ciberseguridad

## Herramientas

<Steps>
1. **Programación:** Python para automatizar procesos, SQL para consultas a bases de datos.
2. **Sistemas operativos:** Linux (código abierto y flexible), macOS y Windows.
3. **Vulnerabilidades web:** Revisar el OWASP Top 10 para conocer los principales riesgos.
4. **Antivirus:** Detecta y elimina software malicioso mediante análisis de memoria.
5. **IDS (Sistema de detección de intrusiones):** Monitorea tráfico y detecta accesos sospechosos.
6. **Encriptación:** Protege la confidencialidad de los datos, impidiendo accesos no autorizados.
7. **Pruebas de penetración:** Simulan ataques para identificar vulnerabilidades en redes, sistemas y aplicaciones.
</Steps>

<Aside type="tip">
Dominar estas herramientas y conceptos es clave para fortalecer la seguridad de cualquier infraestructura.
</Aside>

## Portafolio

Un **portfolio** es esencial para mostrar habilidades, proyectos y conocimientos de forma más profunda que en un currículum. Permite destacarse ante potenciales empleadores.

### Opciones para crear un portfolio

<CardGrid>
  <Card title="Carpeta local" icon="folder">
    Archivos organizados por categorías en tu disco duro.
  </Card>
  <Card title="Google Drive o Dropbox" icon="cloud">
    Permite almacenar y compartir documentos en la nube con actualizaciones en tiempo real.
  </Card>
  <Card title="Google Sites" icon="globe">
    Crear un sitio web profesional, accesible y personalizable.
  </Card>
  <Card title="Repositorio Git" icon="github">
    GitHub, GitLab o Bitbucket para mostrar proyectos, documentación y código en Markdown.
  </Card>
</CardGrid>

### Contenido recomendado

<Steps>
1. Auditorías de seguridad.
2. Análisis de redes.
3. Gestión de permisos en Linux.
4. Consultas SQL.
5. Documentación de incidentes.
6. Revisión de currículum y certificados.
</Steps>

<Aside type="caution">
Evita incluir información privada o protegida por derechos. Mantén tu portfolio en "privado" hasta que esté listo para compartir.
</Aside>
