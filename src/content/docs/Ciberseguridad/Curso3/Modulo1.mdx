# 🌐 ¿Qué es una red?

Una **red** conecta dispositivos para intercambiar datos y compartir recursos: archivos, impresoras, servicios web, cámaras de videovigilancia, etc.  
En casa puede incluir laptops, móviles, televisores inteligentes y consolas de videojuegos; en empresas, servidores, estaciones de trabajo, sistemas de control industrial (ICS/SCADA) y más.

Cada dispositivo necesita **identificadores únicos**:
- **Dirección IP:** indica a dónde enviar los datos.
- **Dirección MAC:** grabada en el hardware de red; se usa para el tráfico local y para aplicar políticas de seguridad (como listas blancas).

<Aside type="note">
Una red no solo conecta máquinas: permite colaboración, automatización y acceso remoto a datos desde cualquier lugar.
</Aside>

- **SAN (Storage Area Network):** red dedicada a discos y servidores de almacenamiento.
- **VPN (Virtual Private Network):** red privada sobre Internet para cifrar y proteger el tráfico.

# 🛠️ Dispositivos clave y su rol

Los datos viajan en **paquetes**, que estos dispositivos procesan:

<CardGrid>
  <Card title="Router" icon="route">
    Conecta redes distintas (ej. red de casa ↔ Internet). Usa tablas de rutas para decidir el mejor camino.
  </Card>
  <Card title="Switch" icon="shuffle">
    Reparte paquetes dentro de la LAN solo al destino correcto; mejora seguridad y rendimiento.
  </Card>
  <Card title="Hub" icon="link">
    Envia los datos a todos los dispositivos. Es simple, barato y antiguo; hoy casi no se usa.
  </Card>
  <Card title="Firewall" icon="shield">
    Inspecciona y bloquea tráfico sospechoso o no autorizado según reglas.
  </Card>
  <Card title="Servidor" icon="server">
    Centraliza recursos: correo, bases de datos, autenticación, copias de seguridad.
  </Card>
  <Card title="AP y módem" icon="wifi">
    AP (Access Point) da Wi-Fi; módem traduce señales de proveedor a nuestra red.
  </Card>
</CardGrid>

## 🔐 Dispositivos avanzados:
- **IDS/IPS:** detectan o bloquean intrusiones.
- **WAF (Web Application Firewall):** filtra ataques web.
- **Load balancer:** reparte tráfico entre varios servidores.
- **Proxy:** actúa como intermediario para ocultar IPs o cachear datos.

# ☁️ Computación en la nube y SDN

## ✅ ¿Qué es?

Permite tener servidores, redes y almacenamiento en centros de datos externos, pagando solo lo usado.  
Ejemplo: Gmail, Google Cloud, AWS o Azure.

## 🧩 Modelos más usados:

<CardGrid>
  <Card title="SaaS (Software as a Service)" icon="app-window">
    Usas apps completas sin preocuparte por servidores. Ejemplo: Slack, Trello, Gmail.
  </Card>
  <Card title="IaaS (Infrastructure as a Service)" icon="database">
    Alquilas servidores virtuales, redes, discos duros. Ejemplo: AWS EC2.
  </Card>
  <Card title="PaaS (Platform as a Service)" icon="code-2">
    Entornos listos para desarrollar apps: base de datos, API, escalado. Ejemplo: Heroku.
  </Card>
</CardGrid>

<Aside>
Hoy muchas empresas combinan nube pública y privada (**nube híbrida**) o usan varios proveedores para evitar fallos (**multicloud**).
</Aside>

## 🌱 Ventajas:
- Escalado casi inmediato.
- Coste inicial menor.
- Alta disponibilidad global.
- Seguridad gestionada por expertos.

## 🧠 SDN (Software Defined Networking):
Permite programar y automatizar redes: cambiar rutas, crear reglas o dividir redes virtuales sin tocar cables.

# 📦 ¿Cómo viajan los datos?

Los datos se dividen en **paquetes** (pequeñas unidades):

- **Encabezado:** información para el envío (IPs, protocolo, TTL).
- **Cuerpo:** datos reales (ej. texto, imagen).
- **Pie:** control de errores.

## 🚦 Ejemplo:
1. Escribes una URL.
2. Tu PC crea paquetes y los envía al router.
3. Routers intermedios deciden la ruta.
4. El servidor responde y tus paquetes vuelven.
5. El navegador reensambla los datos y muestra la web.

<Aside>
Este proceso ocurre en milisegundos, millones de veces al día.
</Aside>

# 📡 TCP/IP: base de Internet

Modelo práctico con 4 capas:

<Steps>
1. **Acceso a red:** hardware físico, Wi-Fi, cables.
2. **Internet:** decide ruta con IP; usa ICMP para notificar errores.
3. **Transporte:** TCP (seguro, ordenado) o UDP (rápido, no garantiza entrega).
4. **Aplicación:** protocolos que vemos: HTTP, FTP, SSH, DNS.
</Steps>

## ⚖️ ¿TCP o UDP?
- **TCP:** email, web, transferencia de archivos.
- **UDP:** streaming, juegos online, videollamadas.

# 🔑 IPv4 vs IPv6

- **IPv4:** 32 bits, direcciones limitadas (~4 mil millones).
- **IPv6:** 128 bits, casi ilimitadas; pensado para IoT, redes móviles y seguridad integrada.

## 🌍 Pública vs privada:
- **Pública:** visible desde Internet.
- **Privada:** solo dentro de una LAN (ej. 192.168.x.x).

---

# 🛣️ Encabezado IPv4 simplificado

Campos como:
- Versión
- Longitud
- TTL (Time To Live)
- Protocolo (TCP/UDP)
- IP origen/destino

**TTL:** evita que paquetes circulen para siempre; baja en cada salto.

# 🛡️ Seguridad en redes

- Firewalls bloquean tráfico no deseado.
- IDS/IPS detectan ataques conocidos o anomalías.
- Monitoreo constante para prevenir fallos.
- Proxy y WAF protegen servicios web.

## 🚨 Principales amenazas:
- DDoS: saturan el ancho de banda.
- Malware y ransomware.
- Phishing para robar credenciales.
- Sniffers o ataques “Man-in-the-Middle”.

<Aside type="tip">
Conocer modelos, protocolos y dispositivos permite construir redes más seguras y resistentes.
</Aside>
