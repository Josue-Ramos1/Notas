# 🔐 Endurecimiento de seguridad

El **Endurecimiento de seguridad** es el proceso de reforzar sistemas y redes para minimizar vulnerabilidades y reducir la superficie de ataque.  
Se compara con asegurar una casa: cerrar puertas y ventanas equivale a aplicar medidas de seguridad en una red.

Este proceso incluye:

- **Mantenimiento regular**: actualizaciones de software, ajustes de configuración y contraseñas seguras.
- **Eliminación de servicios y puertos innecesarios**.
- **Pruebas de penetración (pen test)** para simular ataques y encontrar fallos.
- Restricción de permisos y mejoras en la seguridad física.

---

# 🖥️ Endurecimiento del sistema operativo (OS)

El sistema operativo es clave: un solo error puede comprometer todo.

<Steps>
1. Aplicar **actualizaciones y parches** de inmediato.
2. Crear y mantener una **configuración de línea base** para detectar cambios no autorizados.
3. Eliminar software/hardware innecesarios que puedan tener vulnerabilidades.
4. Implementar **políticas de contraseñas seguras** (longitud, complejidad, cambios periódicos).
5. Activar **autenticación multifactor (MFA)** o **2FA** para reducir riesgos.
</Steps>

<Aside type="note">
Un **ataque de fuerza bruta** es un método de prueba y error para descifrar credenciales. Existen variantes como:
- Ataques simples: prueban combinaciones al azar.
- Ataques de diccionario: usan listas de contraseñas filtradas.
- Ataques automatizados con herramientas especializadas.
</Aside>

---

## 🛡️ Medidas para prevenir fuerza bruta

<CardGrid>
  <Card title="Salting & Hashing" icon="lock">
    Convierte contraseñas en valores cifrados, imposibles de revertir fácilmente.
  </Card>
  <Card title="MFA & 2FA" icon="shield">
    Añaden más pasos de verificación al inicio de sesión.
  </Card>
  <Card title="CAPTCHA / reCAPTCHA" icon="user-check">
    Diferencian humanos de bots, bloqueando ataques automáticos.
  </Card>
  <Card title="Políticas estrictas" icon="settings">
    Contraseñas seguras, bloqueos tras intentos fallidos y cambios periódicos.
  </Card>
</CardGrid>

<Aside type="tip">
Las **máquinas virtuales (VM)** y **entornos aislados (sandbox)** permiten probar software sospechoso sin dañar el sistema.  
Algunos malware detectan estos entornos y cambian su comportamiento para evitar ser detectados.
</Aside>

# 🌐 Endurecimiento de red

El endurecimiento de red protege la infraestructura y limita las oportunidades de ataque.

<Steps>
1. Filtrar puertos y protocolos innecesarios.
2. Mantener actualizaciones y parches.
3. Realizar copias de seguridad.
4. Usar **firewalls**, **IDS**, **IPS** y herramientas **SIEM**.
5. Segmentar la red y aislar zonas críticas.
6. Cifrar la información sensible.
</Steps>

<Aside type="note">
Las herramientas **SIEM** ayudan a analizar registros, detectar patrones y priorizar vulnerabilidades.
</Aside>

---

## 🧩 Defensa en profundidad

Combinación de herramientas para una red más segura:

<CardGrid>
  <Card title="Firewalls" icon="shield">
    Filtran tráfico según reglas definidas; primera línea de defensa.
  </Card>
  <Card title="IDS" icon="activity">
    Detectan amenazas conocidas y envían alertas.
  </Card>
  <Card title="IPS" icon="alert-triangle">
    Detectan y bloquean automáticamente tráfico malicioso.
  </Card>
  <Card title="SIEM" icon="bar-chart-2">
    Centralizan registros de todos los dispositivos para analizar incidentes.
  </Card>
</CardGrid>

---

## 📊 Comparativa de herramientas

| Herramienta | Ventajas | Desventajas |
|--|--|--|
| **Firewall** | Filtra tráfico según reglas | No detecta ataques avanzados |
| **IDS** | Alerta de intrusiones | No bloquea tráfico |
| **IPS** | Bloquea tráfico malicioso | Puede generar falsos positivos |
| **SIEM** | Centraliza datos para análisis | No detiene ataques directamente |

<Aside>
Todas cumplen funciones distintas; juntas forman una estrategia de **defensa en capas**.
</Aside>

# ☁️ Endurecimiento en la nube

La nube ofrece rapidez, escalabilidad y ahorro, pero introduce riesgos nuevos que deben gestionarse.

---

## 🔑 IAM (Identity and Access Management)

Permite definir qué usuarios pueden acceder, a qué recursos y qué pueden hacer.

- Configuraciones erróneas pueden exponer datos críticos.
- Cada cambio debe registrarse y auditarse.

---

## ⚙️ Configuración y superficie de ataque

- Cada servicio activo suma más puntos de ataque.
- Migraciones deben planificarse para evitar brechas.
- Actualizar procesos internos para seguir cambios del proveedor.

<Aside type="note">
El **modelo de responsabilidad compartida**:  
- El CSP protege hardware, red y host.
- El cliente protege configuración, usuarios y datos.
</Aside>

---

---

## 🔒 Gestión de claves

Mantener claves protegidas es esencial.

- **TPM (Trusted Platform Module):** chip físico que guarda contraseñas y claves.
- **CloudHSM:** módulo de hardware para operaciones criptográficas seguras.

<Aside>
Si la clave de cifrado se pierde, ni el proveedor puede recuperarla.
</Aside>

---

## ✅ Certificaciones y auditorías

Los CSP pasan auditorías (p. ej., **FEDRAMP**) que validan buenas prácticas, especialmente en sectores regulados como el gubernamental.

<Aside type="tip">
La responsabilidad final de configurar y proteger datos siempre recae sobre el cliente.
</Aside>

---

# 📌 Resumen final

El endurecimiento de sistemas, redes y nube implica:

- Actualizaciones constantes y parches.
- Configuraciones seguras y líneas base.
- Eliminación de servicios innecesarios.
- Uso de herramientas: firewalls, IDS/IPS, SIEM.
- Segmentación y cifrado.
- Gestión de identidades y claves.
- Revisión periódica, auditorías y pruebas de penetración.

Un enfoque **multicapa** reduce riesgos y fortalece la resiliencia frente a amenazas.
